# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, **please do not open a public GitHub issue.**

Use GitHub's private vulnerability reporting feature: go to the **Security** tab of this repository → **Report a vulnerability** to submit a confidential report directly to the maintainer.

Please include:
- Description of the vulnerability
- Impact assessment
- Reproduction steps (if applicable)
- Suggested mitigation (optional)

## Scope

This policy covers the OpenSOC GitOps Dokploy blueprint itself (Docker Compose definitions, configuration examples, deployment scripts, and documentation). It does not cover vulnerabilities in upstream projects (Wazuh, OpenSearch, Dokploy, Traefik) — please report those directly to their respective maintainers.

## Response

Security reports will be reviewed and handled responsibly. As this is an independently maintained open-source project, response times are best-effort.

Thank you for helping improve the security of the OpenSOC ecosystem.
