# Contributing

Thank you for your interest in OpenSOC-GitOps-Dokploy.

This project aims to provide a production-oriented reference architecture for deploying and operating an Open Source SOC using GitOps principles.

## Ways to Contribute

You can contribute by:

* Reporting bugs
* Suggesting improvements
* Improving documentation
* Sharing deployment experiences
* Proposing security hardening recommendations
* Submitting pull requests

## Before Opening a Pull Request

Please:

1. Open an issue to discuss significant changes.
2. Ensure documentation remains consistent.
3. Avoid committing secrets, credentials, or certificates.
4. Follow the repository structure and conventions.

## Documentation Contributions

Documentation improvements are highly encouraged, including:

* Runbooks
* Troubleshooting procedures
* Architecture Decision Records (ADRs)
* Operational best practices

## Security

For security vulnerabilities, follow the instructions in `SECURITY.md`.

Thank you for contributing.
